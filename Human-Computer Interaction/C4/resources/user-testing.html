User Testing Survey - Public Transport App Prototype

1. Introduction

Hello, and thank you for participating in this user testing session!

Before we begin, I want to highFlight that we are testing the application and not you. There are no right or wrong answers, and your honest feedback is extremely valuable to us. This session will help us understand how to improve public transportation applications to make them more accessible and user-friendly.

This session will take approximately 45-50 minutes, you only need to complete a few tasks using the Accesibility App Prototype application and then complete the survey.

2. Test Case: Accessible Route Planning

Proposed Scenario: "You have a doctor’s appointment in the city center. You use a wheelchair, so you want to avoid stairs and crowded areas. You also need to make sure there are elevators on the route. When you arrive, you find out that an elevator is broken and decide to report the problem using the app."

Tasks to attempt:

Task 1: Start by planning a route from your current location to a destination in the city center (imaginary because the proposed scenario is not real). Try to adjust the route options to match your accessibility needs, such as using a wheelchair, avoiding stairs, and staying away from crowded areas.
Task 2: Review the route options provided. Choose one route and try to check if it includes step-by-step information about accessibility features, such as elevators or ramps.
Task 3: Imagine that you find an accessibility problem on the route. Try to report this issue through the app.

Survey - Complete the following questions based on your experience:

1. Were you able to adjust the route according to your accessibility needs?
2. Was the accessibility information in the route details clear and easy to understand?
3. What type of accessibility information did you see in the route? (Select all that apply)
4. Did you find a way to report an accessibility problem on the route?
5. How would you describe the process of writing and sending the accessibility report?
6. Did the confirmation message after sending your report make you feel that your feedback was received?
7. Did you experience any barriers during this task? (Select all that apply)
What would you improve in the accessible route planning feature?
3. Test Case: Configurable Alerts
Proposed Scenario: "You use public transport every day. You want to get alerts if something changes, like elevator problems or delays. You also want to check old alerts and test if alerts work well on your phone."

Tasks to attempt:

Task 1: Go to the alerts section in the app. Try to choose what kind of alerts you want to get, like problems with elevators or delays in service.
Task 2: Try to change how you receive alerts. For example, make them visual, sound, or vibration.
Task 3: Use the option to send a test alert. Check if the alert appears on your screen.
Task 4: Look at the alert history. Try to find an old alert and read what it says.

Survey - Complete the following questions based on your experience:

1. Were you able to find where to choose the alert types?
2. What alert types did you select? (Select all that apply)
3. Were you able to change how you get the alerts? (Select all that you tried)
4. Did you use the test alert option?
5. Did you find the alert history and read past alerts?
6. Did the alerts make you feel more informed or safe?
7. Did you have any problems with this part of the app? (Select all that apply)
What would you improve in the alert settings or alert history?
