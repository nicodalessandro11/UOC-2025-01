Detected Problems

  • Difficult access to accessibility options: Users found it hard to locate specific accessibility filters.
  • Unclear information: It's not easy to determine if routes have elevators, ramps, or other accessibility features.
  • No reporting mechanism: No way to report when an elevator is out of service or other issues encountered.

Proposed Solutions

  • Prominent filters: Place accessibility options in a visible location with high contrast and adequate size.
  • Clear iconography: Use intuitive icons and descriptive labels for each accessibility feature.
  • Reporting system: Add a button to report accessibility issues during the journey.
  • Detailed information: Specifically show accessibility features for each segment of the route.

2. Configurable Alerts

Objective: Configure personalized alerts to receive notifications about incidents affecting transport accessibility.
1
Open Settings
Settings menu
Accesses the "Alert Settings" section from the main menu of the application.
Curiosity
2
Choose Alert Types
Alert types panel
Selects the categories "Route changes", "Elevator out of service" and "Service disruptions".
Control
3

Notification options
Defines how to receive alerts: visual, sound, vibration, or combinations based on preferences and needs.
Security
4
Test Alerts
"Test alert" button
Uses the function to send a test notification and verify that it works correctly according to the chosen configuration.
Relief
5
Save Configuration
"Save" button
Confirms the chosen preferences and receives confirmation that the configuration has been successfully applied.
Satisfaction

Detected Problems

  • Inaccessible configuration: Users were unable to find where to configure notifications in the application.
  • Limited options: No options to customize the notification method according to specific needs.
  • No testing capability: No way to verify if alerts will work as expected before using them in real situations.
  • No history available: No way to check previous alerts or filter them by category.

Proposed Solutions

  • Prominent access: Add a visible "Alert Settings" button on the main screen.
  • Multimodal alerts: Implement options to receive alerts in visual, auditory, and haptic formats according to preferences.
  • Testing system: Include a button to test each alert type before saving the configuration.
  • Accessible history: Create a section to view and filter previous alerts by category, date, and status.
  • Configurable timing: Allow adjustment of how far in advance to receive notifications.
